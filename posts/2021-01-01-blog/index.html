构建博客的尝试

2021-01-01

读了奈卜拉先生的 我是怎么写博客的，便也想回顾下自己构建博客的一些尝试。感谢奈卜拉先生，您精彩的博文给予我许多帮助。

静态网站生成器

静态网站生成器（static site generator）可以将轻量的标记文本（如 Markdown）转换为标准的标记文本（如 HTML），其中 Hugo 和 Hexo 在博客构建中似乎很流行。

Hexo

我首先尝试了 Hexo，因为网络上 Hexo 的中文教程非常丰富，还有很多酷炫的主题。 但也许是因为 NPM 的依赖管理并不理想，在遇到一个依赖包版本导致生成文档失败的 bug 后，我暂时放弃了它， 前后断断续续尝试了几个月，期间使用了主题 casper。

Hugo

与 Hexo 对比，我感觉 Hugo 构建的速度更快，且 Hugo 对 RSS 提供了良好的支持，内置的模板几乎开箱即用。 我在使用过程中没遇到什么问题，大约尝试了半年，期间使用了主题 Even。

Pandoc

Hexo 和 Hugo 虽然轻量，但定制主题与插件也并不轻松，我开始期望有更轻量的构建方式。

2020 年初偶然发现奈卜拉先生的博客，给我留下了深刻的印象：文章是主体，页面简洁，没有喧宾夺主的元素，没有额外的认知负担。 从 Ta 的博文中，我学习了开源文档转换器 Pandoc 的使用。 写了个脚本用 Pandoc 批量转换 Markdown 为 HTML，抄了一份不错的 CSS 修了修，就是现在的模样了。 这个方式目前已经用了 3 个月。
